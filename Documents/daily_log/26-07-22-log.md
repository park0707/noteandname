## 3. 🐛 이슈 트래킹 및 문제 해결 (Troubleshooting & Resolution)

### [이슈 1] 이미지 드래그 시 마우스를 못 따라오고 고무줄처럼 늘어지는 현상
- **증상 (Symptom)**: 이미지 요소를 마우스로 드래그할 때 커서 반응이 지연되며 고무줄 현상(Rubber-banding) 발생
- **원인 분석 (Root Cause)**: Tailwind/CSS `transition-all duration-200` 스타일이 마우스 좌표가 매 프레임 갱신될 때 200ms 보간을 계속 재설정함
- **해결 방식 (Solution)**: 드래그 중인 요소를 감지하여 `transition: 'none'` 인라인 스타일을 강제 적용함
- **수정 위치**: [WorldMapCanvas.tsx](file:///c:/project/noteandname/src/components/worldview/WorldMapCanvas.tsx)
- **핵심 코드 스니펫**:
```tsx
// ✅ 드래그 시 트랜지션 해제 처리
<g style={{
  transform: `translate(${element.x}px, ${element.y}px)`,
  transition: isDragging ? 'none' : 'transform 0.2s ease-out'
}}>
```

### [이슈 2] 점간 자석 스냅 활성화 시 보라색 스냅 점이 (0,0) 좌측 상단에서 날아오는 잔상 착시
- **증상 (Symptom)**: 자석 스냅 핑(Ping) 인디케이터 생성 시 좌상단 (0,0)에서 순간 이동하는 잔상 발생
- **원인 분석 (Root Cause)**: SVG `<circle>`에 적용된 CSS `animate-ping` (scale keyframe)이 SVG 기본 origin (0,0)을 기준으로 애니메이션을 수행함
- **해결 방식 (Solution)**: CSS `animate-ping` 제거 후 해당 스냅 좌표(`cx, cy`)에 즉시 노출되는 고정 고대비 인디케이터로 수정함
- **수정 위치**: [WorldMapCanvas.tsx](file:///c:/project/noteandname/src/components/worldview/WorldMapCanvas.tsx)
- **핵심 코드 스니펫**:
```tsx
// ✅ 고정 스냅 인디케이터로 변경
{snapPoint && (
  <circle cx={snapPoint.x} cy={snapPoint.y} r={6} className="fill-purple-500 stroke-white stroke-2" />
)}
```
