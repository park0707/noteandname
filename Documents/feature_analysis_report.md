# 소설 창작 지원 도구 필수 기능 분석 보고서
(Novel Creation Support Tools Essential Features Analysis Report)

이 문서는 Novella, Scrivener, Campfire Writing, World Anvil, Novelcrafter, Atticus 등 주요 소설 창작 지원 도구를 분석하여 경쟁력 있는 소설 창작 플랫폼이 갖춰야 할 필수 기능과 한국 웹소설 환경에 특화된 요구사항을 정리한 조사 보고서입니다.

---

## 1. 6대 필수 기능 영역 (Core Features)

### 1-1. 집필 에디터 (Writing Editor)
집필 에디터는 모든 창작 도구의 핵심이며 사용자의 이탈/유지를 결정하는 핵심 요인입니다.
* **집중 모드 (Focus Mode / Distraction-Free)**
  - 화면 전체를 집필 영역으로 전환하고 사이드바와 툴바를 모두 숨김.
  - 어두운 배경색 및 종이 질감의 테마 전환 지원.
  - *사례*: Scrivener, Novelcrafter, Atticus 탑재. Hemingway Editor는 "편집 모드 OFF" 상태에서만 집필을 허용하여 초고 작성과 퇴고를 강제 분리.
* **자동 저장 & 버전 히스토리**
  - *Scrivener*: 사용자 지정 간격(기본 2분) 자동 저장 및 특정 시점 버전을 보관하는 스냅샷(Snapshot) 제공.
  - *Novelcrafter*: 클라우드 실시간 저장 + 회차(Chapter)별 독립 버전 관리.
  - *Atticus*: 자동 저장 + 30일 버전 히스토리 제공.
* **목표 설정 & 집필 통계**
  - 일일 목표 단어 수 설정에 따른 실시간 진행률 바(Progress Bar) 표시.
  - 세션별 집필 속도(분당 단어 수), 총 집필 시간, 연속 집필 일수(Streak) 추적.
  - *사례*: Campfire는 프로젝트 전체 목표 대비 진행률을 대시보드에 표시. NaNoWriMo 연동(외부 API) 도구 존재.
* **맞춤법·문법 검사**
  - 내장 맞춤법 검사기 탑재 또는 LanguageTool API 연동.
  - 한국어 특화 도구의 경우 **부산대 한국어 맞춤법 검사기** 또는 **Naver SpellCheck API** 사용.
  - *사례*: Hemingway Editor는 문장 가독성(Flesch-Kincaid 점수) 및 수동태/부사 과다 사용 경고에 특화.
* **분할 뷰 (Split View)**
  - *Scrivener*: 화면을 상하/좌우로 분할하여 두 문서를 동시에 편집 (예: 왼쪽 이전 챕터, 오른쪽 현재 챕터 표시로 연속성 유지).
  - *Novelcrafter*: 집필 화면 우측에 세계관 패널(캐릭터 프로필, 장소 정보 등)을 슬라이드 오버로 표시.
* **내보내기 (Export)**
  - *Atticus*: EPUB, PDF, DOCX, Print-Ready PDF 단일 클릭 생성 지원.
  - *Scrivener*: 컴파일(Compile) 기능으로 챕터 번호, 헤더/푸터, 폰트를 한 번에 설정.
  - *한국형 특화*: 국내 연재 플랫폼(네이버 시리즈, 카카오페이지, 문피아) 맞춤 형식 지원이 차별화 포인트.

### 1-2. 프로젝트 구조 관리 (Structure Management)
* **코르크 보드 / 카드 보드 (Corkboard)**
  - Scrivener의 대표 기능으로, 각 챕터와 씬을 인덱스 카드(Index Card) 형태로 표시.
  - 카드에는 제목, 시놉시스(요약), 색상 레이블, 상태(초고/수정/완료 등)가 노출됨.
  - 카드를 드래그하여 순서를 재배치하면 실제 문서 순서도 함께 변경됨.
  - 카드 색상을 통해 시점 인물(POV), 장소, 긴장도 등을 시각적으로 코딩(Visual Coding).
* **아웃라이너 (Outliner)**
  - `Act → Chapter → Scene → Beat` 계층 구조의 트리 뷰 표시.
  - 각 노드에 단어 수 목표, 현재 단어 수, 완료 여부 인라인 표시.
  - 행 클릭 시 해당 씬 에디터로 즉각 이동.
* **씬 메타데이터 패널**
  - 각 씬에 배경 장소, 등장인물, 시간대(작품 내 날짜), POV, 요약, 작가 메모 등의 정보 부착.
  - *사례*: Novelcrafter는 씬 패널에서 캐릭터 클릭 시 상세 프로필로 바로 이동하는 하이퍼링크 연결 제공.

### 1-3. 캐릭터 관리 (Character Management)
* **캐릭터 프로필 시트**
  - *기본 필드*: 이름, 나이, 성별, 외모 설명, 직업/역할, 프로필 이미지 업로드(또는 AI 생성 이미지 연동), 한 줄 요약(Tagline), 상세 배경 스토리.
  - *고급 필드 (Campfire, World Anvil)*: MBTI/에니어그램/Big 5 성격 유형 설문, 내적 목표(Want) vs 외적 목표(Need), 두려움, 결함(Flaw), 핵심 신념(Belief), 비밀 항목, 말투 샘플을 기반으로 한 AI 캐릭터 목소리 대사 생성, 첫 등장/마지막 등장 회차 자동 추적.
* **캐릭터 관계도**
  - *Campfire Writing*: 캔버스 위 캐릭터 노드 배치, 엣지(Edge)에 관계 유형 레이블(가족, 연인, 스승-제자, 적대 등) 부착, 관계 강도(1~5) 및 시작 시점 기록, 특정 캐릭터 중심 필터링, PNG/SVG 내보내기 지원.
  - *World Anvil*: 단방향/양방향 관계 구분, 관계의 유효 시간 범위 설정(예: "3장~7장 기간만 동맹"), 조직(가문, 문파, 길드) 노드를 별도 계층으로 추가하여 개인-조직 관계 통합 표시.

### 1-4. 세계관 관리 (World Building)
* **위키형 데이터베이스 (Lore Bible / World Bible)**
  - 항목 유형별 전용 템플릿 제공: 장소(면적, 기후, 지배 세력, 역사 등), 조직(창설 목적, 계층 구조 등), 마법 체계(원리, 한계, 비용 등), 언어(문자 체계, 문법 규칙 등), 사건(날짜, 원인, 결과 등).
  - 항목 간 연결: 다른 항목 이름을 `@`로 멘션 시 자동 하이퍼링크 생성, 역방향 링크 생성("이 캐릭터가 등장하는 씬" 등), 지식 그래프(Graph View) 시각화.
* **타임라인 (Timeline)**
  - 작품 내 시간(픽션 캘린더)과 실제 집필 시간 분리 관리.
  - 이벤트 카드를 타임라인에 배치하고 관련 캐릭터, 장소, 씬 연결.
  - 여러 캐릭터의 행동선을 수평 레인(Swimlane)으로 분리해 동시 진행 상황 파악.
  - 작품 내 "현재 날짜" 설정 시 미래/과거 이벤트 자동 구분.

### 1-5. 플롯 & 복선 관리 (Plot & Foreshadowing)
* **플롯 보드 (Plot Board)**
  - 카드 기반 칸반 보드 형태로 발단-전개-위기-절정-결말 컬럼에 배치.
  - 각 카드에 관련 챕터, 캐릭터, 주제(Theme) 태그 연결.
  - 순서 변경 시 "영향받는 챕터 경고" 표시.
* **복선 추적 시스템 (Foreshadowing)**
  - "복선 카드" 생성: 복선 내용, 심는 위치(씬/챕터), 회수 위치 입력.
  - 복선 상태 관리: 심음(Planted) → 발전(Developed) → 회수(Paid Off) 추적.
  - 미회수 복선 경고 대시보드: 작품 완성 전 회수하지 않은 복선 목록 자동 표시.
  - 씬과 연결 시 해당 씬 에디터 사이드 패널에 복선 카드 노출.

### 1-6. 생산성 & 협업 도구 (Productivity & Collaboration)
* **검색 & 레퍼런스**
  - 전체 검색(Global Search): 프로젝트 내 모든 문서, 캐릭터 프로필, 세계관 항목 단일 검색. 특정 단어가 등장하는 씬 목록 즉시 조회.
  - 메모 & 리서치 보관함: 참고 자료(이미지, 링크, 텍스트)를 첨부하여 집필 화면 분할 뷰로 열어두고 참고.
* **멀티 디바이스 동기화**
  - 브라우저 기반 동기화(Novelcrafter, Atticus) 또는 Dropbox/iCloud 동기화(Scrivener). 오프라인 집필 지원 여부가 장기 사용자 유지의 핵심.
* **목표 & 습관 트래킹**
  - 일간/주간/월간 목표 설정, 연속 집필 일수(Streak) 카운터, 목표 달성 배지/보상 시스템(게이미피케이션), 뽀모도로 타이머(집필 세션) 지원.

## 3. 한국 웹소설 창작 도구 추가 요구사항

글로벌 도구들이 대응하지 못하는 한국 웹소설만의 특화 요구사항은 다음과 같습니다.
1. **회차 (Episode) 중심 구조**
   - 글로벌 도구의 "챕터" 개념과 달리 한국 웹소설은 "회차" 단위로 연재됨.
   - 회차별 분량 가이드(2,000~3,500자), 클리프행어(Cliffhanger) 체크리스트, 회차 단위 업로드 기능이 필수적임.
2. **플랫폼 직접 연동**
   - 네이버 시리즈, 카카오페이지, 문피아 등 주요 플랫폼 API 연동을 통한 원고 직접 업로드.
   - 플랫폼별 금칙어 필터링 및 성인 등급 분류 자동화.
3. **장르별 클리셰 경고**
   - 헌터/회귀/빙의/무협/로판 등 장르별 빈출 설정 및 플롯 패턴 데이터베이스 구축.
   - 기존 유명 웹소설과 유사한 캐릭터 이름 및 설정 검출 시 경고.
4. **독자 반응 대시보드**
   - 조회수, 선호작 등록수, 댓글 추이를 집필 도구 내 대시보드에서 통합 확인.
   - "이탈률이 높은 회차/씬" 데이터와 원고 내용을 나란히 비교 분석할 수 있는 도구 제공.

## 4. Novelflow 개발 우선순위 제안

* **즉시 필요 (우선순위 1)**
  - 코르크보드 / 아웃라이너 (구조 관리용)
  - 씬 메타데이터 패널 (캐릭터 및 세계관 연결 고리)
  - 집필 통계 + 목표 단어 수 트래커
* **단기 (우선순위 2)**
  - 캐릭터 관계도
  - 복선 추적 대시보드 (Campfire, Novelcrafter와의 차별화 요소)
  - 전체 검색 (Global Search)
* **중기 (우선순위 3)**
  - 세계관 위키 + 하이퍼링크 (World Anvil의 핵심 기능 간소화 버전)
  - 타임라인 연동 (회차 단위 웹소설 전용)
  - 회차 단위 내보내기 (플랫폼 연동의 첫 단계)
* **장기 (우선순위 4)**
  - 가상 세계 지도
  - 가상 언어 생성기
  - 플랫폼 직접 업로드 API

## 5. 외부 문서 가져오기 (Import) 기능 기획 명세 (구현 예정)

웹소설 작가들이 외부 플랫폼이나 개인 클라우드 스토리지에서 집필하던 원고 및 설정 자료를 노벨플로우 집필실 내부로 간편하게 가져올 수 있도록 하는 기능의 기획 사양입니다.

### 5-1. Google Drive 연동 가져오기
- **인증 및 연동**: OAuth 2.0 프로토콜을 사용해 사용자의 Google 계정 인증 및 드라이브 읽기 권한 획득.
- **파일 탐색기**: Google Picker API를 활용해 웹소설 집필실 내에서 드라이브 내 폴더 및 파일(`.txt`, `.docx`, `.gdoc`)을 탐색하는 모달 팝업 제공.
- **텍스트 변환 파이프라인**:
  - `.txt`: UTF-8 인코딩 기준으로 일반 텍스트 추출 후 새 에피소드 문서로 삽입.
  - `.docx` / Google Docs: 파일 내 볼드, 이탤릭 등의 기초 서식 및 줄바꿈을 HTML 단락(`<p>`) 구조로 자동 마이그레이션하여 원고 구조 훼손 방지.

### 5-2. Notion 데이터베이스 및 페이지 가져오기
- **연동 방식**: Notion 통합(Integration) 토큰을 사용하여 작가의 노션 워크스페이스에 대한 접근 허가.
- **페이지 파싱 및 마크다운 변환**:
  - 지정한 노션 페이지나 데이터베이스 내의 문서 카드를 마크다운 형식으로 읽어옴.
  - 마크다운의 `#`, `##` 등의 헤딩은 에디터의 제목 블록으로 변환하고, 하이퍼링크 및 본문 텍스트는 리치 텍스트 형태로 노벨플로우 문서 목록 내에 회차별 문서로 매핑하여 생성.
- **구조적 임포트**: 노션 데이터베이스의 '회차 번호', '작성일' 프로퍼티를 읽어 노벨플로우 에피소드 순서 및 메타데이터 필드에 자동 바인딩.

## 6. 버전 스냅샷 자동화 및 Split Diff 비교 아키텍처

소설 집필 중 발생할 수 있는 원고 소실 및 임의 수정 실수를 방지하고, 이전 창작 시점의 원고와 비교 분석하기 위한 복원 및 비교 엔진 구조입니다.

### 6-1. 다원적 자동 스냅샷 수집 엔진
- **시간 기준 스케줄러**: 에디터 활성화 상태 시 10분 주기로 자동 스냅샷(`auto_time`)을 무음 백그라운드 생성하여 안전장치 마련.
- **분량 변동 트리거**: 이전 저장 시점 대비 글자 수 차이가 ±1,000자에 도달할 때 자동 스냅샷(`auto_words`)을 생성하여 대규모 텍스트 변경 이벤트 기록.
- **용량 최적화**: 단일 회차당 최대 50개의 스냅샷 히스토리 슬롯을 제한(LIFO 구조)하여 브라우저 로컬 스토리지 한도 초과 오류 방지.

### 6-2. Split Diff 시각화 모듈 (LCS 알고리즘)
- **분석 알고리즘**: 단어 단위 최장 공통 부분 수열(LCS, Longest Common Subsequence) 비교를 실행하여 텍스트의 미세 변화 추적.
- **사용자 경험(UX) 분할 디자인**:
  - **좌측 뷰 (현재 원고)**: 기준 스냅샷 대비 추가된 단어는 연초록 배경(`bg-green-500/20 text-green-500`), 삭제된 단어는 연빨강 배경/취소선(`bg-red-500/20 line-through`)으로 매크로 하이라이팅 표시.
  - **우측 뷰 (스냅샷 원고)**: 선택된 스냅샷의 온전한 텍스트 상태를 읽기 전용으로 시각화.
- **동작 방식**: 비교 뷰에서 작가가 변경 사항을 세밀히 대조한 후, "복원 승인" 클릭 시 에디터 상태를 스냅샷 상태로 되돌리고 `lastSnapshotWordCount`를 동기화하여 연속적인 자동 저장을 연계 지원.

### 6-3. 스냅샷 복원 시 에디터 DOM 비동기화 이슈 및 해결 과제 (미구현)
- **현상**: 스냅샷 복원 시 내부 데이터 상태(React State)는 업데이트되나, 브라우저의 리치 텍스트 에디터(`contentEditable`) 내부 DOM 구조 및 Selection 포커스가 즉시 동기화되지 않는 간헐적 복원 무반응/실패 문제가 존재합니다.
- **조치 현황**: 현재 버전에서는 복원 승인 프로세스를 진행해 에디터에 임시 매핑하지만, 근본적인 동기화 오류는 미구현으로 분류하고 추후 에디터 컴포넌트 생명주기 마운트 갱신 훅을 보강해 보완 예정입니다.
- **해결 로드맵**:
  - `contentEditable` 에디터의 리렌더링 키(Key) 강제 갱신 기법 도입.
  - Undo/Redo 히스토리 스택과의 가상 DOM 매핑 정렬.
