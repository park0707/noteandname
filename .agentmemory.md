## Key Architectural Decisions
- **2026-07-06 WritingSpace Modularization**: Separated the massive 2,300-line `WritingSpace.tsx` orchestrator into modularized subcomponents under `src/components/workspace/writingspace/` (`EpisodeSidebar`, `EditorToolbar`, `FindReplaceBar`, `DiffViewPane`, `SplitEditorPane`, `MainEditorCanvas`, and `Modals/`). Reduced code complexity, file size, and state coupling.
- **2026-07-06 Recursion Traversal Deduplication**: Consolidated multiple separate recursive child tree-element loops in deletion, trash-moving, and restore logic into a unified utility function `getRecursiveDescendants` in `utils.ts`.
- **2026-07-15 Alert/Confirm Replacement**: Replaced all native browser `alert()` and `confirm()` calls with a customized promise-based `AlertConfirmContext` and `useAlertConfirm` hook. Completely removed confirm dialogs when moving documents to trash since they can be easily restored anyway.
- **2026-07-16 WorldMap Multi-Selection & Element Dragging**: Implemented multi-selection and dragging of map elements in WorldMap select mode (`editMode === 'select'`). Grouped elements recursively under group nodes (type `'group'`). When selecting or dragging a grouped element, the entire group is chosen and moved as a unit. Marquee dashed-border drag select selects all elements within the marquee bounding box. Group deletion restores nested child elements to their map-level parent map ID instead of deleting them. Dynamic cursor styling (e.g. `cursor-move` on draggable boundaries, `cursor-crosshair` for drawing, `cursor-help` for measurement) was also added.

## Lessons Learned & Workarounds
- **FontFaceSet `add` Method Type Error**: In certain TypeScript environments, standard typings for `FontFaceSet` might lack the `add` method, causing compiler errors when calling `document.fonts.add(...)`. 
  - **Workaround**: Cast to `any`, i.e., `(document.fonts as any).add(fontFace)`.
- **JSX Type Namespace Conflicts**: Explicitly annotating React rendering helper functions with `JSX.Element[]` can cause compiler failures depending on global React/TS configuration differences. Allow TypeScript to infer these return types automatically by omitting explicit signatures.
- **Typewriter Mode Layout Shift (Focus-Induced Scrolling)**: When typewriter mode is active, browser-native cursor tracking might programmatically scroll the outer layout wrapper or stretch it based on the editor's virtual scroll heights, shifting absolute elements like the sidebar fold toggle button.
  - **Workaround**: Force explicit height and width bounds (`h-full w-full`) on the outer `WritingSpace` flex container, and isolate `EpisodeSidebar` inside a dedicated container wrapper (`<div className="h-full shrink-0 relative flex">`). This anchors the absolute positioned button relative to a fixed-height, non-scrollable layout column, securing its vertical position.
- **Toolbar Dropdown Clipping (overflow-x-auto Container)**: The editing toolbar contains scrollable items (`overflow-x-auto`) to fit smaller screens. However, any container with horizontal scrolling clips vertical absolute positioned children (browsers force `overflow-y: hidden/scroll`).
  - **Workaround**: Render dropdowns (such as the Font Family and Divider dropdowns) with `position: fixed` instead of `absolute`, and compute their viewport coordinates dynamically on open using `.getBoundingClientRect()`. This completely bypasses container clipping.
- **Promise-based Custom Alert/Confirm Modal Dialogs**: Browser native dialogs block thread execution, freeze rendering, and violate modern style guidelines. A theme-compliant promise-based `AlertConfirmProvider` context was wrapped at the app root level. It uses a MutationObserver to dynamically toggle light/dark styles based on class additions on document element, providing high-fidelity custom popups.
- **Canvas Middle-Click Panning Workaround**: Windows/Chrome browsers automatically activate circular autoscroll when the scroll wheel (e.button === 1) is clicked. React's synthetic `onMouseDown` does not reliably cancel this behavior, causing dragging/panning to lock up.
  - **Workaround**: Register a native `mousedown` event listener directly on the container reference (`canvasContainerRef`) and execute `e.preventDefault()` if `e.button === 1` to bypass autoscroll.
- **SVG 타일 격자선 모아레(Moiré Pattern) 현상 원인 및 해결**: `shape-rendering="crispEdges"`는 안티앨리어싱을 강제 Off 처리하여 모니터 픽셀 올림/내림(Rounding)을 수행한다. 이 상태에서 지도를 멀리 축소(Zoom out)하거나 줌 배율이 소수점일 경우, 화면상 격자 간격이 3px, 4px로 무작위 매핑되며 어지러운 직물 체크 패턴(Moiré)이 발생함.
  - **Workaround**: 
    1) `shapeRendering="geometricPrecision"`을 지정하여 부드럽고 매끄러운 선 렌더링을 보장한다.
    2) **Level of Detail (LOD) 동적 스케일 조절**: `effectiveGridSize * zoom < 14` 일 때 격자 간격을 2배, 4배, 8배로 자동 확대한 `effectiveGridSize`를 대입한다. 이로써 줌아웃을 하든 축소를 하든 화면상 격자 밀도가 14px 미만으로 과밀해지지 않아 **모아레 패턴이 물리적으로 100% 발생하지 않는 무한 격자**를 구현한다.
- **Strict Scope Modification & Workflow Rules**:
  1. **Plan first**: Review relevant specifications in `Documents/` and existing codebase design before creating a plan.
  2. **Efficiency & Reuse**: Check `utils.ts` and hooks for reusable code to prevent duplication.
  3. **Identify targets**: Map exact target lines and scope boundaries before edits.
  4. **Preserve user modifications**: Keep user-edited files, comments, and incomplete hooks intact unless they cause build-breaking errors. Report user-edited areas in completion summaries.
  5. **Compile verification**: Always run `npx tsc -b` and `npm run lint` to verify code health before completing tasks.
- **Git Commit Message Style**: 커밋 메시지는 반드시 **한글**로 작성한다. 형식은 진행한 작업의 핵심 내용을 담은 **한 줄 요약** (예: `"세계관 지도 사이드바 접기/펼치기 기능 추가"`). 영문 커밋은 사용하지 않는다.
- **Open Source Library Usage Rule**: 어떤 기능을 구현할 때 무조건 제로 베이스에서 직접 만들지 말고, 오픈 소스 라이브러리 중 상업적 이용이 가능한 무료 라이브러리가 있는지 먼저 조사하여 활용한다. 만약 라이브러리에 저작권자 표기(Attribution) 등 특정 조건이 붙어 있다면, 반드시 구현 전에 사용자에게 사전 고지하고 승인을 얻어야 한다.
