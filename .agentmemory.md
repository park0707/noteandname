## Lessons Learned & Workarounds
- **FontFaceSet `add` Method Type Error**: In certain TypeScript environments, standard typings for `FontFaceSet` might lack the `add` method, causing compiler errors when calling `document.fonts.add(...)`. 
  - **Workaround**: Cast to `any`, i.e., `(document.fonts as any).add(fontFace)`.
- **JSX Type Namespace Conflicts**: Explicitly annotating React rendering helper functions with `JSX.Element[]` can cause compiler failures depending on global React/TS configuration differences. Allow TypeScript to infer these return types automatically by omitting explicit signatures.
- **Typewriter Mode Layout Shift (Focus-Induced Scrolling)**: When typewriter mode is active, browser-native cursor tracking might programmatically scroll the outer layout wrapper or stretch it based on the editor's virtual scroll heights, shifting absolute elements like the sidebar fold toggle button.
  - **Workaround**: Force explicit height and width bounds (`h-full w-full`) on the outer `WritingSpace` flex container, and isolate `EpisodeSidebar` inside a dedicated container wrapper (`<div className="h-full shrink-0 relative flex">`). This anchors the absolute positioned button relative to a fixed-height, non-scrollable layout column, securing its vertical position.
